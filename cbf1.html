搭博客少走弯路-在线管理Hexo站点

前言

关于管理博客所使用的工具，其实我已经换过很多次了，从最开始的系统自带记事本+cmd窗口，还是Typora破解版+自己写的bat批处理，还是挺麻烦的，后来逛Github时，看到一个跟Hexo有关的插件，名叫hexo-admin，便赶紧搞到了本地的环境上，经过一通鼓捣之后，终于是实装到了网站上（现在这篇文章就是在hexo-admin上编辑的）

初始化

  1. 安装hexo-admin插件并在本地环境运行
  2. 进入本地环境，网址http://localhost:4000/admin
  3. 点击上方导航栏的Settings，在下方图片所标记区域，从上到下，依次填写名称，密码，加密cookies用的密码（这个随便填），然后将Admin Config Section下的代码粘贴到_config.yml内

Posts页面

  • 点击文章标题，右边会显示文章内容，双击文章标题会转到编辑页面，左边为Markdown格式的文章源代码，右边为源代码渲染后的效果

  • 点击New Post，会自动弹出文章标题输入框，输入完成后按下回车可进入新文章编辑页面，导航栏下方标题和文章源代码可编辑，右上角设置按钮可设置文章属性，点击Publish按钮会自动将此文章从_drafts（草稿）移动至_posts（文章），Publish按钮变为Unpublish，再点一次，则进行相反操作

  • 文章会自动保存，当意外关闭网页时，文章不会丢失

  • 当试图往文章源代码输入框粘贴图片时，会自动将图片移至博客/img/目录下，这一点根据Setting里的设置而定

Pages页面

大致使用方法和Posts页面一样，但只有创建了目录Pages，名称才会正确识别

About&Deploy页面

About: 记录了关于hexo-admin插件和Hexo作者的主页，起不到太大作用

Deploy: 部署页面的额外方法（似乎是），现有指令可满足正常Deploy需求，这里暂不介绍

Settings页面

  • 设置密码的部分已经写进初始化那一段里了

Editor Settings（编辑器设置）

  • Enable line numbering.：给文章原文件编辑区的左边添加每一行的序号

  • Enable spellchecking. (buggy on older browsers)：启用拼写检查（不支持老式浏览器）

Image Pasting Settings（粘贴图像时的设置）

  • Always ask for filename：试图在文章原文件编辑区内粘贴图像时询问图片文件名

  • Overwrite images if file already exists：试图在文章原文件编辑区内粘贴图像时覆盖原有文件

  • Image directory：试图在文章原文件编辑区内粘贴图像时，图片文件所在的目录

  • Image filename prefix：试图在文章原文件编辑区内粘贴图像时，图片文件名称的前缀
