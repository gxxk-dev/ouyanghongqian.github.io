GxxkSystem自述文件/帮助文档


这是一个专门针对mPython（掌控板）而开发的系统，您需要一块版本至少为2.0的掌控版才能体验其基础功能

此介绍文件基于GxxkSystem1.1.22_230117-B版本编写

此项目仓库地址：https://gitee.com/ouyanghongqian/gxxk-system/

特点

  • 不使用图形化进行编辑，针对性能方面进行优化（当然，也会有更多的bug…）
  • 联网获取插件（或应用），免去了官方更新缓慢导致第三方App不能快速正式上线的问题
  • 屏幕空间利用率大，少了操作提示，上升了学习成本，却提高了效率
  • 代码含有大量注释和Debug日志（就是那堆print），便于用户二次开发
  • 在Gitee开源，可提交Pull Requests（合并请求，简称PR）和Issue（可理解为反馈Bug）
  • 使用开源协议 MIT 开源，方便修改（本来想用WTFPL的，毕竟和我系统名字很合得来，不是吗，笑）
  • 可联网更新，方便操作

插件规范

一个可以被GxxkSystem正常加载的插件应该具有以下功能：

  1. 一个名为start的函数，而且此函数可启动该插件
  2. 一个名为info的列表，而且此列表格式如下
    1. 第一项是插件的名称，不能太长
    2. 第二项是插件的作者名称，也不能太长
    3. 返回主界面的入口
      同时，此插件不能含有读写文件的部分，如需要保存用户数据，可使用GxxkSystem/pluginApi.py下的功能

关于插件API

GxxkSystem为了方便开发者的插件开发，将部分函数的用法的说明文档开放给开发者使用，可对照下面的表使用对应函数

安装

新版本的.mpy文件除了文件扩展名不一样和boot/main保持原文件以外，其他的对应的功能都是一样的

新版安装

下载发布页的GxxkSystemInstaller.mpy，刷入到mPython内（保持原文件名），然后创建文件：main.py，写入：import GxxkSystemInstaller 重启后进入安装程序，按A安装，完成后重启体验GxxkSystem

旧版安装

如需要手动安装，直接将Gitee当前版本目录下的内容刷入即可，但是更新必须刷入config，否则有可能导致某项配置不存在发生错误

首先，刷入安装程序GxxkSystemInstaller.py，刷入后重启掌控版，按A确认安装，然后将开始写入GxxkSystem的系统文件，完成后重启就看到内容了

使用方法

掌控版（后面缩写为mpy）通电后，将进入加载界面，约0.5s左右，之后将进入主菜单界面（mpy屏幕最上方文字为当前界面标题）

在主菜单情况下，有4种功能的入口，分别是：

  • 在线插件，按下P（或Y）进入
  • 本地插件，按下T（或H）进入
  • 设置，按下O（或N）进入
  • 锁屏功能，按按钮A进入

接下来，将介绍这4个功能的用法以及作用

锁屏功能

将在屏幕左边绘制一个时钟（圆心x32y32，半径32像素），右边绘制4行文本（1,4行x60，2,3行x64）

按按钮A回到主界面

设置

在设置菜单内，有1种功能的入口，分别是：

  • WiFi设置，按下P（或Y）进入
  • 更新选项，按T（或H）查看进入方法并在3s后重启
  • 关于页面，按O（或N）查看

此菜单内，按除了上述所说的按键以外的所有按键均可退出当前界面

WiFi设置

在WiFi设置下，共有3个选项，分别为：

  • WiFi信息设置，按下P（或Y）进入
  • 连接WiFi，会根据上一次设置的WiFi信息进行连接，按下T（或H）进入
  • 同步时间，会以东八区，使用cn.ntp.org.cn服务器进行时间校准，按下O（或N）进入

此菜单内，按除了上述所说的按键以外的所有按键均可退出当前界面

更新系统

由于其功能特殊性质，需要特殊操作才可触发，一旦执行，所有配置信息均被重置（如WiFi名，WiFi密码等）同时，此功能需要联网下载数据，所以请在更新前设置好WiFi信息

进入方法：重启后长按A键，直到出现正在更新的提示就算触发，在此功能下，将会自动连接WiFi并联网获取系统安装包，下载完成后又会自动重启，重启后按A键开始安装，完成后就自动进入新版本了

  • Y：将光标向下移动一格，如移动前就已经是最后一个，那么会将光标移动到第一个

  • A：进入当前插件的详细页面

此菜单内，按除了上述所说的按键以外的所有按键均可退出当前界面

GxxkTyper

GxxkTyper，中文名Gxxk打字机，此组件可输入26个大小写字母，0-9的数字和英文标点符号，下面是操作流程

进入后，将出现4行文字，分别是：

ABCDEFGHIJKLMNOP
QRSTUVWXYZ
0123456789
标点符号（这里省略）

我们需要通过PYTHON中的PYTH来进行操作，其中，第一行就是P，第二行就是Y，以此类推，想输入哪个就先点击对应的类，点了之后又会出现这个类细分的4个选项，以此类推，直到每个类都只有一个字符的时候，再点击要输入的字符，就成功输入了这个字符了

同时，剩下的A B O N 4个按钮也有不同的功能，分别是：

  • O：显示当前已输入的字符，再按一次返回
  • N：切换为大写模式（如果已经是大写模式，那么将切换为小写模式）
  • A：在最开始的输入界面按下时完成输入
  • B：在最开始的输入界面按下时退格

小技巧：同时，如果你的mPy可以连接电脑，那么可以在成功连接上之后按下Control+C，然后输入想输入的内容，此时你在mPy已输入的内容就被修改成了你输入的内容

版本命名规则

Beta版本

此版本用于测试，一般主体功能和UI已经成型，但是还是含有一些Bug

示例：1.4.12.230101-B

主版本1，4是Beta版本的推出次数，如果发布Release版本就重置，修改12项，在2023年1月1日编译，版本测试版

Release版本

用于正常使用的标准版，几乎没有Bug

示例：1.4.5.4_230102-R

主版本1，LTS版本4，第五个Release版本，合并了4个Beta版本，在2023年1月2日编译，版本发布版

LTS（长期支持）版本

长期版本，在下个LTS版本还没有发布的时候均可在帖子下面询问

示例：1.5.5_230103-LTS

主版本1，包含4-5的所有功能，一共合并了5个Release版本，230103，版本LTS版
